These instances are from the main, weighted, anytime track of the MaxSAT 2024 competition.

FORMAT: WCNF (2022+ format)

The instances use the new WCNF format (post-2022). Key differences from the old format:
- There is NO p-line (no "p wcnf ..." header).
- Hard clauses start with `h`, followed by literals, terminated by `0`.
- Soft clauses start with their integer weight, followed by literals, terminated by `0`.
- Comment lines start with `c`. Each file has a JSON comment header with instance statistics (nvars, ncls, clause length stats, weight stats, etc.).

Example hard clause:  h 3 -5 7 0       (must be satisfied)
Example soft clause:  42 -1 3 0         (weight 42, adds 42 to cost if falsified)

Some instances have no hard clauses at all (nhards: 0) — in that case every clause is soft.

COST DEFINITION:

A valid solution is an assignment of all variables to true/false that satisfies ALL hard clauses.
The cost of a solution = sum of weights of all FALSIFIED soft clauses.
The goal is to minimize this cost.
A solution that violates any hard clause is invalid.

INSTANCE STATISTICS:

The comment header at the top of each .wcnf file contains useful metadata in JSON format:
- nvars: number of variables
- ncls: total number of clauses
- nhards / nsofts: number of hard / soft clauses
- nhard_len_stats / nsoft_len_stats: clause length statistics
- nsoft_wts: number of distinct soft clause weights
- soft_wt_stats: weight distribution statistics (min, max, avg, stddev)

best-costs.csv: one row per instance. Columns: instance, best_cost. A best_cost of -1 means no known solution.
